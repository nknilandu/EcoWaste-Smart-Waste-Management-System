Build a modern full-stack smart waste management web application called “EcoWaste” using React + TypeScript + Tailwind CSS on the frontend and Supabase for authentication, database, realtime notifications, and role-based access control.

This is a 3-role system:
1. Citizen
2. Collector
3. Admin

The app should feel production-quality in UI/UX, but keep the code modular, clean, and easy to maintain. Use a responsive dashboard-based layout with a polished landing page.

==================================================
CORE PRODUCT GOAL
==================================================
Create a Smart Waste Management System where:
- Citizens can monitor nearby waste bins by area, submit complaints, complete daily waste disposal tasks, view notifications, and manage their profile.
- Collectors can view assigned collection tasks, mark tasks in progress, complete collections, and update waste collection status.
- Admins can manage waste bins, users, complaints, collectors, and collection schedules from a central dashboard.

Use Supabase as the main backend and structure the app to work with existing tables and relational logic.

==================================================
SUPABASE DATABASE STRUCTURE
==================================================
Use the following tables and relationships:

1. profiles
- id (uuid, primary key)
- user_id (references auth.users.id)
- full_name (text)
- avatar_url (text)
- area (text)
- created_at (timestamp)
- updated_at (timestamp)
- email (text)

2. user_roles
- id (uuid, primary key)
- user_id (references auth.users.id)
- role (text) -> citizen, collector, admin

3. bins
- id (uuid, primary key)
- location (text)
- latitude (numeric)
- longitude (numeric)
- fill_level (int4)
- status (text) -> empty, half_full, full, overflow, under_maintenance
- sensor_id (text)
- area (text)
- subarea (text)
- created_at (timestamp)
- updated_at (timestamp)

4. collections
- id (uuid, primary key)
- bin_id (uuid, references bins.id)
- collector_id (uuid, references auth.users.id or collector user id)
- pickup_time (timestamp)
- status (text) -> pending, in_progress, completed, missed
- created_at (timestamp)
- updated_at (timestamp)

5. complaints
- id (uuid, primary key)
- user_id (uuid, references auth.users.id)
- bin_id (uuid, references bins.id)
- details (text)
- status (text) -> pending, reviewing, resolved
- created_at (timestamp)
- updated_at (timestamp)

6. notifications
- id (uuid, primary key)
- user_id (uuid, references auth.users.id)
- title (text)
- message (text)
- is_read (bool)
- type (text)
- created_at (timestamp)

7. daily_tasks
- id (uuid, primary key)
- user_id (uuid, references auth.users.id)
- bin_id (uuid, references bins.id)
- otp (text)
- completed_at (timestamp)

8. disposal_logs
- id (uuid, primary key)
- user_id (uuid, references auth.users.id)
- bin_id (uuid, references bins.id)
- disposed_at (timestamp)

==================================================
AUTHENTICATION & ROLE LOGIC
==================================================
Implement Supabase authentication with:
- Login page
- Registration page for citizen and collector only
- Admin accounts are not self-registered; they are seeded or manually assigned

After login:
- Detect the user role from user_roles
- Redirect citizens to /citizen-dashboard
- Redirect collectors to /collector-dashboard
- Redirect admins to /admin-dashboard

Protect all dashboard routes:
- Only authenticated users can enter protected pages
- Only the correct role can access its dashboard and internal routes
- If role does not match, redirect to login or unauthorized page
- Never default to the wrong role silently

- Assigned collections list:

BUSINESS RULES
==================================================
- Citizen sees bins based on their area
- Collector sees only their assigned collection tasks
- Admin sees all data
- Complaint submission creates a citizen notification
- Complaint resolution creates a citizen notification
- Completing a collection resets bin fill level to 0
- Daily task completion is recorded per user and bin
- Admin cannot be self-registered on public registration page
